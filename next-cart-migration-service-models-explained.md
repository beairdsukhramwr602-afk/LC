# Next-Cart Migration Service Models Explained

Next-Cart offers three service models for migration projects: Standard Service, Managed Service, and Custom Service. Each model supports the same broad goal of moving store data from a Source Cart to a Target Cart, but they differ in who performs the migration process and whether the project requires customization or modification work.

Understanding these differences helps customers avoid two common mistakes: choosing a service only by price, or assuming that every more complex project simply means Next-Cart performs everything from start to finish. The safer choice starts with a clearer understanding of what each service actually covers.

### The Three Service Models at a Glance <a href="#the-three-service-models-at-a-glance" id="the-three-service-models-at-a-glance"></a>

The three service models answer different questions.

| Service model    | Main meaning                                                                                                        | Best understood as                   |
| ---------------- | ------------------------------------------------------------------------------------------------------------------- | ------------------------------------ |
| Standard Service | The customer performs the migration using the Next-Cart migration tool with purchased Add-ons.                      | Customer-led execution               |
| Managed Service  | Next-Cart performs the migration process for the customer using default tool capability and purchased Add-ons.      | Next-Cart-led execution              |
| Custom Service   | Next-Cart handles customization, modification, tailored Add-ons, custom Add-ons, or bespoke migration requirements. | Customization and exclusive handling |

The service models are related, but they are not simply three levels of the same support package. Standard Service and Managed Service mainly differ in execution responsibility. Custom Service is different because it is triggered by customization or modification needs.

### Standard Service <a href="#standard-service" id="standard-service"></a>

Standard Service is the customer-led service model.

With Standard Service, the customer owns a 1-year migration license for the selected Source Cart and Target Cart pair. The customer uses the Next-Cart migration tool to perform the migration for that platform pair.

Standard Service includes:

* access to the migration tool for the selected platform pair
* customer-led migration execution
* 24/7 expert support
* optional standard Add-ons where needed
* the ability to review, re-run, and continue migration activity within the license terms and available Entity Points capacity

Standard Service is usually the most suitable model when the customer is comfortable operating the migration process, reviewing the result, and handling configuration decisions within the default tool capability.

### What Standard Service Does Not Mean <a href="#what-standard-service-does-not-mean" id="what-standard-service-does-not-mean"></a>

Standard Service does not mean the customer is unsupported. Next-Cart support is available to help clarify technical questions, configuration issues, and migration concerns.

It also does not mean the project cannot use Add-ons. Standard Add-ons can be purchased with Standard Service when the customer needs focused help with filtering, advanced mapping, or data configuration.

The key point is execution responsibility. With Standard Service, the customer performs the migration process.

### Managed Service <a href="#managed-service" id="managed-service"></a>

Managed Service is the Next-Cart-led execution model.

With Managed Service, the customer also owns a 1-year migration license for the selected Source Cart and Target Cart pair. The key difference is that Next-Cart’s experts perform the migration process for the customer.

Managed Service includes:

* Next-Cart-led migration execution
* configuration of the migration tool within the default capability
* integration of purchased standard Add-ons
* reduced customer execution burden
* customer review of the migrated result
* support for the migration process within the selected service scope

Managed Service is useful when the migration can be handled through the default tool capability and purchased Add-ons, but the customer wants Next-Cart to perform the process instead of managing the execution internally.

### What Managed Service Does Not Mean <a href="#what-managed-service-does-not-mean" id="what-managed-service-does-not-mean"></a>

Managed Service does not automatically include customization or modification work.

If the project requires tailored Add-ons, custom Add-ons, migration-tool modification, Custom Cart handling, third-party app or extension data handling, custom fields, outside-system identifiers, or other bespoke configuration, the project requires Custom Service.

Managed Service is about who performs the migration process. Custom Service is about whether the project requires customization or modification work.

### Custom Service <a href="#custom-service" id="custom-service"></a>

Custom Service is the service model for customization, modification, exclusive handling, and bespoke migration requirements.

Custom Service is required whenever the project needs work beyond the default migration tool's capabilities or the standard Add-on's capabilities.

Custom Service can include:

* tailored Add-ons
* custom Add-ons
* migration-tool modification
* Custom Cart handling
* third-party app, plugin, module, or extension data
* custom fields
* outside-system identifiers
* platform capability limitations that need tailored handling
* special transformation rules
* bespoke configuration required to preserve the expected outcome

Custom Service is not used only because a project is large. It is used when the project needs custom or exclusive handling to produce an acceptable target result.

### Custom Service and Migration Management <a href="#custom-service-and-migration-management" id="custom-service-and-migration-management"></a>

Custom Service should not be confused with Managed Service.

Custom Service means customization or modification work is required. It does not automatically mean Next-Cart performs the entire migration process for the customer.

A Custom Service plan can work in more than one way:

* the customer may purchase customization work and still perform the migration process themselves
* the customer may include optional migration management so Next-Cart performs and manages the migration process
* the final plan may include both customization work and Next-Cart-led execution when that is the agreed service arrangement

This distinction matters because a Custom license status reflects the presence of customization work. It does not always define who performs the migration execution.

### Add-ons Across the Three Service Models <a href="#add-ons-across-the-three-service-models" id="add-ons-across-the-three-service-models"></a>

Standard Add-ons can be purchased with Standard Service, Managed Service, or Custom Service.

The current standard Add-ons are:

* Data Filter Add-on
* Advanced Data Mapping
* Advanced Data Configure

When a standard Add-on is used within its default capability, it can remain part of the selected service model.

For example:

* a Standard Service customer can use the Data Filter Add-on and still perform the migration themselves
* a Managed Service customer can purchase standard Add-ons and have Next-Cart integrate them into the migration process
* a Custom Service customer can include standard, tailored, or custom Add-ons as part of a broader custom plan

The service impact changes when an Add-on needs modification beyond the default capability.

### When Add-ons Move a Project Into Custom Service <a href="#when-add-ons-move-a-project-into-custom-service" id="when-add-ons-move-a-project-into-custom-service"></a>

If a standard Add-on needs to be modified beyond its default capabilities, the modification work is handled through Custom Service.

If the customer needs a custom Add-on that is not currently available, that request is also handled through Custom Service.

This applies because any customization or modification work belongs under Custom Service.

For a Standard Service customer, the license status can change to Custom when modified Add-on work is added. The customer may still perform the migration themselves unless they also include optional migration management.

For a Managed Service customer, the customized Add-on is integrated into the migration run handled by Next-Cart as part of the agreed service arrangement.

### Pricing Difference Between the Services <a href="#pricing-difference-between-the-services" id="pricing-difference-between-the-services"></a>

The service models also affect how pricing is structured.

At a planning level:

* Standard Service uses the Entity Points Plan plus optional Add-ons
* Managed Service uses the Entity Points Plan plus optional Add-ons plus the service fee
* Custom Service uses a custom quote because the work depends on the required customization, entity volume, tailored or custom Add-ons, migration-tool adjustment, and any optional migration management included in the final plan

This pricing structure reflects the different kinds of work involved. Standard Service is customer-led execution. Managed Service adds expert-led execution. Custom Service is priced around the specific customization or modification work required.

### Customer Responsibility Across All Service Models <a href="#customer-responsibility-across-all-service-models" id="customer-responsibility-across-all-service-models"></a>

The execution burden changes across service models, but customer reviews remain important in every model.

The customer should still confirm whether the migrated result is acceptable for launch. That includes reviewing:

* product buying behavior
* customer and order usability
* key category or browse paths
* content and URL-sensitive pages where relevant
* Add-on-influenced results
* Custom Service outputs where applicable
* Recent Data Migration results before go-live

Next-Cart can support, perform, or customize migration work depending on the service model and final plan. The business still needs to confirm that the target store supports the expected outcome.

### How the Services Work Together in a Real Migration <a href="#how-the-services-work-together-in-a-real-migration" id="how-the-services-work-together-in-a-real-migration"></a>

A migration project can move from one service status to another when the work changes.

A customer may begin with Standard Service and later discover that an Add-on needs modification. The customization work moves the license status to Custom, but the customer may still perform the migration if migration management is not included.

A customer may choose Managed Service because they want Next-Cart to perform the migration, then add a modified Add-on later. The license status changes to Custom because the project now includes customization work, and the modified Add-on is integrated into the Next-Cart-led migration run.

A customer may begin directly with Custom Service when they already know the project requires custom Add-ons, Custom Cart handling, migration-tool modification, or bespoke configuration.

The service model should reflect the work the migration actually needs, not only the first assumption made before review.

### Conclusion <a href="#conclusion" id="conclusion"></a>

Standard Service, Managed Service, and Custom Service describe different kinds of migration involvement. Standard Service is customer-led execution. Managed Service is Next-Cart-led execution using default capability and purchased standard Add-ons. Custom Service is the path for customization, modification, tailored Add-ons, custom Add-ons, Custom Cart handling, migration-tool adjustment, and other bespoke requirements.

The most important distinction is that Managed Service and Custom Service answer different questions. Managed Service is about who performs the migration process. Custom Service is about whether the migration requires customization or modification work.

Review your migration needs through both questions: who should perform the migration, and does the project require work beyond the default capability? If the answer is unclear after Demo Migration, or if Add-ons, Custom Cart, custom fields, or third-party data are involved, Live Chat can help clarify the right service status before you purchase or upgrade.

### FAQs <a href="#faqs" id="faqs"></a>

#### What are the three Next-Cart service models? <a href="#what-are-the-three-next-cart-service-models" id="what-are-the-three-next-cart-service-models"></a>

The three service models are Standard Service, Managed Service, and Custom Service.

#### What is Standard Service? <a href="#what-is-standard-service" id="what-is-standard-service"></a>

Standard Service is the customer-led model. The customer uses the Next-Cart migration tool to perform the migration for the selected Source Cart and Target Cart pair.

#### What is Managed Service? <a href="#what-is-managed-service" id="what-is-managed-service"></a>

Managed Service is the Next-Cart-led execution model. Next-Cart performs the migration process for the customer using default tool capability and any purchased standard Add-ons.

#### What is Custom Service? <a href="#what-is-custom-service" id="what-is-custom-service"></a>

Custom Service is the model for customization, modification, tailored Add-ons, custom Add-ons, migration-tool adjustment, Custom Cart handling, or other bespoke migration requirements.

#### Does Custom Service always mean Next-Cart performs the migration? <a href="#does-custom-service-always-mean-next-cart-performs-the-migration" id="does-custom-service-always-mean-next-cart-performs-the-migration"></a>

No. Custom Service means customization or modification work is required. Next-Cart performs the migration only when migration management is included in the final plan.

#### Can Add-ons be used with every service model? <a href="#can-add-ons-be-used-with-every-service-model" id="can-add-ons-be-used-with-every-service-model"></a>

Yes. Standard Add-ons can be purchased with Standard Service, Managed Service, or Custom Service.

#### When does an Add-on make the project Custom? <a href="#when-does-an-add-on-make-the-project-custom" id="when-does-an-add-on-make-the-project-custom"></a>

An Add-on moves the project into Custom Service when it needs modification beyond default capability or when the customer requests a custom Add-on.

#### Does Managed Service include custom work? <a href="#does-managed-service-include-custom-work" id="does-managed-service-include-custom-work"></a>

Not by default. Managed Service means Next-Cart performs the migration process using default capability and purchased standard Add-ons. Customization or modification work requires Custom Service.

#### Does the customer still need to review the result with Managed Service or Custom Service? <a href="#does-the-customer-still-need-to-review-the-result-with-managed-service-or-custom-service" id="does-the-customer-still-need-to-review-the-result-with-managed-service-or-custom-service"></a>

Yes. The customer should still confirm whether the migrated result is acceptable for launch, especially where business-critical behavior, Add-ons, or Custom Service outputs are involved.
